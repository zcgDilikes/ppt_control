代码 Bug 报告(error.txt)
==========================

扫描范围:整个项目 ~9200 行 Python 源码
  - pc_gesture/    (6 文件,1447 行)
  - ppt_core/      (10 文件,2306 行)
  - ppt_qt/        (15 文件,2479 行)
  - ppt_pc_client.py (1 文件,2803 行)

扫描方式:三个子 agent 并行读文件 + 主 agent 串行分析 + pytest 全套测试。
测试结果:**191 个测试全部通过** — 也就是说这些 bug 都是测试覆盖不到的运行时问题。

严重度图例:
  [CRITICAL] 用户可见功能失效 / 安全风险 / 数据丢失
  [MAJOR]    稳定性 / 性能 / 资源问题,可能在特定场景崩溃
  [MINOR]    边界 / 死代码 / 小问题,通常无即时影响

============================================================
一、CRITICAL 级 bug(必修)
============================================================

[1] OPEN_PPT 创建 0 字节 .pptx 文件,PowerPoint 打不开
    文件: ppt_core/ppt_executor.py:363-412 (_open_ppt)
    现象: tempfile.mkstemp(suffix=".pptx") 创建 0 字节空文件
          os.startfile 把空文件交给 PowerPoint 失败
          失败被外层 except Exception: pass 吞掉
    触发: 手机端发 OPEN_PPT 且没带 path 字段(gesture_bridge 写死 path="")
    修复: 用 python-pptx 创建真实空演示文稿
          或 comtypes.client.CreateObject("PowerPoint.Application") 后保存

[2] SCREENSHOT 同秒并发覆盖 + 阻塞鼠标线程 200ms
    文件: ppt_pc_client.py:876
    现象: 文件名 f"screen_{int(time.time())}.png" 精度只到秒
          同秒内连发两条 SCREENSHOT 写同一文件 + 卡线程
    触发: 手机端快速点两次截屏
    修复: 改 time.time_ns() 或加锁串行化

[3] execute_command 同步执行 SCREENSHOT,鼠标线程阻塞
    文件: ppt_pc_client.py:816, 873-884
    现象: pyautogui.screenshot() 在 mouse_render_thread 同步执行 (~50-200ms)
          阻塞期间 LASER/MOUSE_CLICK 全部排队
    触发: 截屏时持续激光
    修复: 移到后台线程,完成后再 push SCREENSHOT_DONE

[4] LASER 缺 dx/dy 时走绝对定位,鼠标瞬移 + 反复跳点
    文件: ppt_pc_client.py:987-995
    现象: dx is None and dy is None 时把整个 dict 序列化进 latest_msg
          mouse_render_thread 读 data.get("x")/("y") 绝对定位
    触发: 手势模块发 {"cmd":"LASER","x":0.5,"y":0.5} 无 dx/dy
    修复: 显式三分支:有 dx/dy 走累加,有 x/y 走绝对,都没有则忽略

[5] WS 服务端 LASER/MOUSE_CLICK 消息无 roomId 校验
    文件: ppt_pc_client.py:1137-1142 (websocket_client_loop)
    现象: 只对 MINI_HELLO/ONLINE/OFFLINE 做了 room 校验
          LASER/MOUSE_CLICK 直接 dispatch,跨房间污染
    触发: 同 SpringBoot 实例内 A/B 房间错位转发
    修复: 入 LASER/MOUSE_CLICK 前 _presence_room_id_matches 校验

============================================================
二、MAJOR 级 bug(应修)
============================================================

[6] pynput mouse Controller.position 在 Windows 是只读
    文件: ppt_core/mouse_controller.py:78, 92
    现象: pynput 在 Windows 上 controller.position = (x, y) 抛 AttributeError
          set_absolute 路径完全失效,只能走 pyautogui
          pyautogui.FailSafeException 也被 except 吞掉
    修复: 改用 ctypes + win32api.SetCursorPos,或累加 dx/dy 逼近

[7] GetObject(class=) 在 PPT 未运行时**创建**隐藏 PPT 实例
    文件: ppt_core/ppt_executor.py:78-94 (_ppt_show_view)
    现象: _wc.GetObject("PowerPoint.Application") 会创建空 PowerPoint.exe
          每次触发 FULL_SCREEN 留一个幽灵进程
    修复: 改用 _wc.GetActiveObject(..., result=...)(只获取不创建)

[8] command_dispatcher 持锁期内执行 COM,所有命令阻塞
    文件: ppt_core/command_dispatcher.py:73-74
    现象: with self._lock: self._dispatch_locked(d) 把派发+执行都上锁
          GetObject 卡住时,激光笔 dx/dy 全部排队
    修复: 锁只保护"读 dict + 选 cmd",实际 execute 用 ThreadPoolExecutor 派出去

[9] reload_config 未被 try/except 包裹(唯一漏点)
    文件: ppt_core/gesture_bridge.py:233-234 (swap_roles)
    现象: 其他 callback 都 except Exception: pass,这里漏了
    修复: 加 try/except

[10] requests.get(...) 成功路径未显式 close,连接池易耗尽
     文件: ppt_core/downloads.py:197-204
     现象: resp 进入 with open 块,iter_content 完成时 GC 回收
           但 with open 抛 IOError 时,resp.close() 不会被调用
     修复: 外层 try/finally: resp.close() 或 contextlib.closing(resp)

[11] ws_client backoff 上限 30s 永远到不了,卡在 16s
     文件: ppt_core/ws_client.py:74-77, 240-246
     现象: delay = min(delay * 2.0, 16.0) 永远 ≤ 16
           wait_s = min(delay, 30.0) 也 ≤ 16
           30s 常量是死代码
     修复: 删 30.0 或把 _BACKOFF_CAP_S 调到 ≥ 30

[12] VERSION_MISMATCH fire-and-forget 任务未持有引用
     文件: ppt_core/ws_client.py:290-294
     现象: asyncio.ensure_future(ws.send(...)) 没存 task 引用
           event loop 关闭时 task 被 GC,警告永远发不到手机
     修复: task 存 self._pending,gather 等待

[13] _dispatch_locked 持锁执行 COM,详见 [8]

[14] os.startfile 非 Windows 平台崩溃
     文件: ppt_qt/app.py:211-217
     现象: macOS/Linux 上 os.startfile 不存在,AttributeError 被 except 吞
     修复: 按 sys.platform 分支处理

[15] 私有属性穿透 GestureBridge._on_status/_on_fps
     文件: ppt_qt/app.py:75-76
     现象: 直接赋值私有属性,未来改 Signal 会静默失效
     修复: 加 set_status_callback / set_fps_callback 公共方法

[16] 退出时不 join WsClient / GestureBridge / PptNotesWorker
     文件: ppt_qt/app.py:301-313 (_quit_app)
     现象: self._ws.stop() 后立即 self._win.close()
           asyncio 线程 / worker 还在跑,C++ 对象已 delete 后 emit
           偶发 RuntimeError: wrapped C/C++ object of type X has been deleted
     修复: 逐个 stop() + join(timeout)

[17] QImage 不复制 buffer,引擎线程覆写时预览花屏
     文件: ppt_qt/pages/gesture_page.py:432-468
     现象: QImage(data, w, h, format) 不复制,直接引用 snap.frame_rgb
           engine 线程下一帧覆写 buffer,QImage.scaled() 期间读到撕裂内存
     触发: 启动手势后预览
     修复: QImage(bytes(snap.frame_rgb), ...).copy() 强制拷贝

[18] QFrame.__dict__ 注入属性
     文件: ppt_qt/pages/gesture_page.py:254, 258, 642
     现象: row.__dict__["_action_lbl"] = ... 在某些 PySide6 6.5+ 被 GC 丢弃
     修复: 改 self._cheat_row_labels: Dict[str, QLabel] = {}

[19] 双高亮路径 race condition(2s 内重复触发会闪烁)
     文件: ppt_qt/pages/gesture_page.py:541-562
     现象: _update_sync_highlight 与 _poll_bridge_gestures 都用
           setStyleSheet + QTimer.singleShot(2000, clear)
           第二次触发会覆盖第一次的清除时刻,导致高亮闪一下就灭
     修复: QTimer 实例成员保存"清除时刻",新一次高亮前先 stop 旧 timer

[20] 教学引导 _last_seen_ts 推进策略错失 target
     文件: ppt_qt/pages/gesture_tutorial_dialog.py:158-172 (_poll_recognition)
     现象: new 里有非 target 项时也把 _last_seen_ts 推到最新
           后续 target 项 ts < last_seen_ts,永远不被处理
     修复: 只在 target 命中时才推进 last_seen_ts

[21] stopwatch reset() 不重启 timer
     文件: ppt_qt/overlays/timer_overlay.py:103-119
     现象: stopwatch 模式 reset() 无参,只清零不重启
           用户期望"重置并继续走表",实际是"归零不计时"
     修复: stopwatch reset 时 self._timer.start()

[22] LASER 灵敏度双重放大(6x 手感灾难)
     文件: ppt_pc_client.py:33, 1037
     现象: pending_laser_dx/dy 已是乘过 sens 的 delta
           PC 端再 * LASER_SENS 二次放大
           手机 sens=2 → 实际 2*6=12 倍跳
     修复: 与移动端约定明确(PC 不再乘,或手机只传未乘 delta)

[23] _ppt_notes_warn_slideshow_once 全局 once 永不重置
     文件: ppt_pc_client.py:143-154, 516-533
     现象: _notes_warned_slideshow = True 后永不重置
           关闭 PPT 再开 → 不再提示
     修复: 检测到 SlideShowWindows.Count >= 1 时重置

[24] presence_room_id_matches 高频 ONLINE/OFFLINE 抖动让 GUI 闪烁
     文件: ppt_pc_client.py:1130-1136
     现象: 配对标题只翻 true/false,无去抖
     修复: 加 500ms 去抖窗口

[25] download_file 不解析 Content-Disposition,basename 路径穿越

     现象: file_name = os.path.basename(file_url.split("?")[0])
           恶意 URL "../../etc/passwd" 越界写

[26] mouse_render_thread 异常被 except: pass 吞掉,无任何日志
     文件: ppt_pc_client.py:1014-1051
     现象: execute_command 异常 print 也不打,调试极难

[27] settings 写文件期间未持锁,JSON 内容可能混合状态
     文件: ppt_pc_client.py:114-124 (_write_settings_file)
     现象: 多线程并发 set_client_settings 时写文件内容不一致

[28] _quit_app 主线程 time.sleep(0.3) 卡 Tk mainloop
     文件: ppt_pc_client.py:2771
     现象: 0.3s 内 Tk 不处理事件,GUI 无响应

[29] pystray tray_thread 未 join,关闭后 mainloop 残留 RuntimeError

[30] merge_remote_client_settings 中 bool("false") == True

[32] QTimer.singleShot 闭包持有已 destroy 的 widget

[33] 灵敏度 spinbox 拖动时每次 valueChanged 同步写 JSON
